# 三套场景演示操作手册

> 本文档配套 `演示资料/01_客户上传演示_三套场景/` 使用。
> 演示前请确认：浏览器打开 `http://127.0.0.1:8888/`，右上角顶栏可看到「客户预审 / 管理端」切换器。

## 0. 演示前检查（30 秒）

### 0.1 知识库状态确认（已就绪 ✅）

知识库当前已导入 **7 个法规 source / 7 chunk / 7 vector**，embedding 走 UniAPI `text-embedding-v4`，可直接跑预审。

| 来源 | 司法辖区 | 用途 |
|---|---|---|
| 企业物质禁忌矩阵演示规则 | GLOBAL | 物料不相容判定 |
| 企业内部化工准入红线演示规则 | GLOBAL | 内部红线物质筛查 |
| 企业储运兼容性演示规则 | GLOBAL | 储运分区与配载 |
| 企业氧化剂工艺安全演示规则 | GLOBAL | 氧化剂工艺风险 |
| 中国危险化学品目录演示摘录 | CN | 危化品识别 |
| ECHA REACH/SVHC 演示摘录 | EU | 受限物质筛查 |
| EPA TSCA 与 OSHA HCS 演示摘录 | US | 美国市场准入 |

> 如要现场重新演示「导入知识包」动作：顶栏切换到「管理端」→ `#/admin/kb` → 点「清空」（输入 `CLEAR` 二次确认）→ 再点「导入演示包」即可看到 7 个 source 重新加载的过程。

### 0.2 顶栏视角

- **客户预审**（默认）：演示客户侧上传 → 预审 → 看报告的完整闭环
- **管理端**：演示知识库管理、Trace 审查回放（D44 演示能力）

## 1. 核心选项速查表

### 1.1 五种「预审场景」对应

| 场景代码 | 中文名 | 默认勾选的检查项 | 用在哪 |
|---|---|---|---|
| `market_access` | 市场准入预审 | 资料完整性 / 成分识别 / 禁限用筛查 / SDS 关键章节 / 工艺适配 / 储运 / 目标市场法规 | **场景 A** 最常用，目标市场准入 |
| `substitution` | 替代物料评估 | 上面 7 项 - 法规筛查 + 相容性 | 替代物料评估 |
| `supplier_intake` | 供应商资料准入 | 资料完整性 / 成分识别 / 禁限用 / SDS 关键章节 / 供应商一致性 / 人工复核 | **场景 C** 最合适，资料盘点为主 |
| `process_introduction` | 工艺导入风险评估 | 资料完整性 / 成分识别 / **相容性** / 工艺适配 / 储运 / 人工复核 | **场景 B** 最合适，含相容性 |
| `storage_safety` | 储运与现场安全评估 | 资料完整性 / 成分识别 / 相容性 / 储运 / SDS / 人工复核 | 仓库/现场环节 |

### 1.2 目标市场（多选）

- **中国 (CN)**：触发危化品目录、内部红线
- **欧盟 (EU)**：触发 ECHA SVHC
- **美国 (US)**：触发 EPA TSCA、OSHA HCS
- 「全球 (GLOBAL)」：触发内部规则

**演示建议**：场景 A 全选三大市场展示「多市场合规」能力；B、C 可只选 CN 节省时间。

## 2. 场景 A · 合规通过（水基清洁剂）✅

**演示目标**：走通顺路径，5 步流程一气呵成，最终判定 `pass / 可审`，可下载 PDF 报告。

**资料目录**：`演示资料/01_客户上传演示_三套场景/A合规通过_水基清洁剂/`
- `SDS_安全数据表.txt`
- `配方表.txt`
- `工艺说明.txt`

### 操作流程

| 步骤 | 路由 | 操作 | 看点 |
|---|---|---|---|
| **进入** | `http://127.0.0.1:8888/#/cases` | 右上角点「＋ 新建 Case」 | Case 看板有历史 case 卡片 |
| **Step 1** | `#/cases/new?step=1` | • 标题：`水基清洁剂_准入演示`<br>• 预审场景：**市场准入预审**<br>• 目标市场：勾选 **CN + EU + US** | pbar 第 1 段亮 |
| **Step 2** | `?step=2` | 把 `A合规通过_水基清洁剂/` 下 3 个 txt 全部拖入 dropzone | dropzone 三件套上传完成，进度条到 100 % |
| **Step 3** | `?step=3` | 资料预检 4 张 metric 卡：已识别 3 / 需补 0 / 可直接检查 N / 受限阻断 0 | **无 alert-card**，CTA「已知悉，继续」 |
| **Step 4** | `?step=4` | 审查范围 18 项，**保持默认勾选**（场景已自动推荐 7 项），点「确认范围」 | stepbar 显示「已选 7 / 共 18」 |
| **Step 5** | `?step=5` | 点中央「**运行预审**」Pill | runcard 6 子任务行依次点亮，~30-60 s 完成 |
| **报告** | `#/cases/<新id>` | 自动跳转到报告页 | • Hero **绿色 dot · 可审**<br>• 状态条：3 chip 全绿<br>• 不合规清单：「无不合规事项」气泡<br>• 右上「下载 PDF 报告」可下到 186 KB PDF |

### 演示话术

> 「这是市场准入场景的标准走通路径——3 份资料齐全、成分合规、无禁限用物质。系统对接 ECHA / EPA / 国内目录 3 个司法辖区做匹配，最终给出『可审』判定，客户可直接进入下一环节。整个预审过程不超过 1 分钟。」

## 3. 场景 B · 不相容阻断（氧化剂 + 易燃物）⛔

**演示目标**：触发企业不相容矩阵红线，预审判定 `not_approved / 不可审`，报告页显示 4 件套不合规清单 + 抽屉看证据。

**资料目录**：`演示资料/01_客户上传演示_三套场景/B不相容阻断_氧化剂加易燃/`
- `SDS_安全数据表.txt`（含强氧化剂 + 易燃溶剂双组分）
- `配方表.txt`
- `工艺说明.txt`

### 操作流程

| 步骤 | 操作 | 看点 |
|---|---|---|
| **进入** | 顶栏切到「客户预审」→ `#/cases` → 「＋ 新建 Case」 | — |
| **Step 1** | • 标题：`氧化剂易燃配方_工艺风险演示`<br>• 预审场景：**工艺导入风险评估**（关键：含 `compatibility_risk` 检查项）<br>• 目标市场：勾选 **CN**（或全选） | 场景下拉切到「工艺导入」 |
| **Step 2** | 上传 `B不相容阻断_氧化剂加易燃/` 下 3 个 txt | dropzone 三件套上传完成 |
| **Step 3** | 资料预检通过（3 份完整） | 无 alert-card |
| **Step 4** | 保持默认勾选 6 项（已自动包含**物料相容性与危险组合**），点「确认范围」 | 已选 6 / 共 18 |
| **Step 5** | 点「运行预审」 | runcard 跑约 30-60 s |
| **报告** | 自动跳报告页 | • Hero **红色 dot · 不可审**<br>• Alert-strip：触发禁忌矩阵<br>• 4 件套清单第一项标红：「氧化剂 + 易燃溶剂同体系，违反企业禁忌矩阵」<br>• 点「查看证据」→ **右侧抽屉滑出** |

### 抽屉证据演示（D44 卖点）

报告页右上角抽屉按钮 / 点不合规条目「查看证据」：
- **客户视角**：只显「规则原文 + 触发原因」
- **管理员视角**（顶栏切「管理端」→ 报告页重 mount）：抽屉额外解锁 3 个 admin tabs：
  - **RAG 证据**：检索命中的法规 chunk + vector 分数 + rerank 分数
  - **Agent 分支**：物料 / 工艺 / 储运 / 法规 4 个子 Agent 的分支推理
  - **规则匹配**：触发的具体规则 ID + 完整文本

### 演示话术

> 「这套配方包含强氧化剂和易燃溶剂双组分，属于工艺导入场景的高风险组合。系统的『相容性 Agent』命中了企业禁忌矩阵，判定为不可审。下面我们切到管理端，可以看到完整的 RAG 检索链路和 Agent 分支推理——每个判断都有规则原文支撑，便于复核和申诉。」

## 4. 场景 C · 资料不全（缺工艺说明项）⚠️

**演示目标**：演示资料预检闸门——上传后 Step 3 直接拦截，**不进入正式预审**，引导客户补件。

**资料目录**：`演示资料/01_客户上传演示_三套场景/C资料不全_缺工艺说明/`
- `SDS_安全数据表.txt`
- `配方表.txt`
- `工艺说明_有缺项.txt`（工艺关键字段缺失）

### 操作流程

| 步骤 | 操作 | 看点 |
|---|---|---|
| **进入** | `#/cases` → 「＋ 新建 Case」 | — |
| **Step 1** | • 标题：`未知中间体_资料盘点演示`<br>• 预审场景：**供应商资料准入**（关键：含 `intake_readiness` + `manual_review`）<br>• 目标市场：勾选 **CN** | 场景下拉切到「供应商资料准入」 |
| **Step 2** | 上传 `C资料不全_缺工艺说明/` 下 3 个 txt | 3 份都能上传成功 |
| **Step 3** | **关键看点**：资料预检 metric 卡显示「**需补 ≥ 1**」 | • 出现 **alert-card 黄底**<br>• 列出缺失字段（如「工艺说明缺少 XX 段」）<br>• 显示「上传补件 →」按钮回到 Step 2<br>• CTA 文案改为「**已知悉，继续**」（允许带瑕疵前进） |
| **演示分支 ①（推荐）** | 点「上传补件 →」回 Step 2 → 演示补件能力 | 模拟客户回头补资料 |
| **演示分支 ②** | 点「已知悉，继续」→ Step 4 → Step 5 | 演示带缺项强行预审的判定结果（通常会被 `manual_review` 标为 `needs_supplement`） |
| **报告** | 跳报告页 | • Hero **黄色 dot · 需复核/补件**<br>• 不合规清单：「⚠️ 工艺说明字段缺失，建议补充后重审」<br>• Sidebar 该 Case 状态色变黄 |

### 演示话术

> 「这种『资料看起来齐 3 份、实际有关键字段缺失』的情况在供应商资料准入场景非常常见。系统在 Step 3 资料预检阶段就给出黄牌警告，明确告诉客户『哪份资料缺什么字段』，并提供回头补件的入口。我们也允许客户在了解风险后强行继续，但报告里会被标记为『需补件』状态，避免漏判和误判。」

## 5. 演示串场建议（顺序）

按「合规 → 阻断 → 补件」三层递进：

1. **A 合规**（3 分钟）：展示完整闭环 + PDF 下载 → 建立信任感
2. **B 阻断**（4 分钟）：展示 Agent 推理 + 抽屉证据 → 体现 RAG 价值（**主秀**）
3. **C 补件**（2 分钟）：展示预检闸门 + 补件交互 → 体现产品对真实业务流的覆盖

中间穿插 30 秒切到「管理端」展示：
- `#/admin/kb` 知识库 7 条 source 一览（合规可控）
- `#/admin/cases/<场景B的id>/trace` 审查回放（10 节点全链路，Agent 分支细节）

---

## 6. 兜底应急方案

| 卡点 | 应对 |
|---|---|
| Step 5 跑预审超过 90 s 没回 | 查看 uvicorn 日志，可能 LLM（DeepSeek）超时；按 F5 重新进入会自动续跑 |
| 报告页白屏 | console 看错误，多数是 `case.latest_report` 为 null，回 Case 看板重选 |
| 知识库突然变 0 chunk | 顶栏切管理端 → `#/admin/kb` → 点「导入演示包」 |
| PDF 下载 404 | 该 Case 没跑过预审；先回 Step 5 跑完再下 |
| 想清场重来 | 管理端 → 找到该 Case 卡片 → 删除（或调 `DELETE /chemical/cases` 清全部） |

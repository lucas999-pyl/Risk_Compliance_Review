## 4 个演示 Case · 通用 Wizard 操作

每个 Case 都走 Wizard 5 步流程。**Step 2 / Step 3 / Step 5 三步操作相同**，只 Step 1（标题）和 Step 4（上传文件）按 Case 切换。

### Step 2 · 基础信息（4 个 Case 都一样）

| 字段 | 值 |
|---|---|
| 物料类型 | 涂料 |
| 目标市场 | ☑ CN ☑ EU ☑ US |
| 预期用途 | 工业涂装 |

### Step 3 · 预审场景 + 检查项（4 个 Case 都一样）

**预审场景（review_scenario）**：`市场准入预审`

**检查项（check_types）勾这 7 项**：

| 勾选 | 检查项 | 说明 |
|---|---|---|
| ☑ | 资料完整性与可审性 | SDS / 配方 / 工艺三件套是否齐 |
| ☑ | 成分识别与 CAS/浓度完整性 | 配方组分 CAS 抽取与浓度 |
| ☑ | 禁限用物质与红线物质筛查 | REACH SVHC / 危化品目录 / 企业红线 |
| ☑ | SDS 关键章节核查 | GB/T 16483-2008 16 章结构 |
| ☑ | 工艺条件适配性 | 温度 / 压力 / 关键步骤 |
| ☑ | 储存与运输条件核查 | SDS 第 7 章 / UN 编号 |
| ☑ | 目标市场法规匹配 | EU REACH / US TSCA / CN GB |

### Step 5 · 跑预审 → 看报告

点「开始预审」→ 自动跳转报告页。

## 演示中可能被问到的问题 · 备答

**Q1：规则原文从哪里来？**
- 内置 7 条企业演示规则 + 7 份官方标准摘要（GB 30981 VOC 限值 / GHS-CLP-GB 30000.2 标签 / OSHA HCS / ECHA SVHC / EPA TSCA / GB 应急管理部 2015 危化品目录 / 企业禁忌矩阵）
- 入库路径：`POST /chemical/knowledge/import-demo-pack` + 官方原文 manifest upload
- 当前状态：14 sources / 21 chunks / 21 vectors

**Q2：报告里的"用户原文"段落是真的从我上传文档抽的吗？**
- 是。工具用规则关键词在文档里高亮锚定，截取前后 ~80 字符窗口
- 演示 3 case_not_approved 的 VOC finding 是最直观的证据

**Q3：为什么有的规则名还是英文？**
- 多数已透传中文名 + GB 标准号副标题
- 极少数（如 `formula_components_missing` / `sds_missing_sections`）因 sub-X rules 数组与后端硬编码 rule_id 命名错位，留在英文 fallback。这是已知数据问题，不影响 verdict 正确性

**Q4：能上传 PDF 吗？**
- 当前 demo 资料是 `.md`（markdown），后端实际支持 `.txt` / `.md` / `.pdf`（可读 PDF 走 pypdf 抽文本，不可读 PDF 报"机器不可读"）

## 重置演示状态（多次演示之间用）

如需把演示库清空重来：

```bash
# 清向量库
curl -X DELETE http://127.0.0.1:8888/chemical/knowledge

# 重新入 A 路径（内置 7 条规则）
curl -X POST http://127.0.0.1:8888/chemical/knowledge/import-demo-pack

# 重新入 B 路径（官方 7 份原文）
cd /home/oneus/projects/化工合规工具/Risk_Compliance_Review
python -c "
import json, urllib.request
from pathlib import Path
BASE='http://127.0.0.1:8888'
PACK=Path('data_samples/chemical_knowledge_sources/official_pack_2026_05')
manifest=json.loads((PACK/'manifest.json').read_text(encoding='utf-8'))
def post(p,d):
    req=urllib.request.Request(BASE+p,data=json.dumps(d).encode(),headers={'Content-Type':'application/json'})
    return json.loads(urllib.request.urlopen(req).read())
for src in manifest['sources']:
    fn=src['filename']
    content=(PACK/fn).read_text(encoding='utf-8')
    payload={k:v for k,v in src.items() if k!='filename'}
    created=post('/knowledge/sources', payload)
    post('/knowledge/ingest', {'source_id': created['id'], 'content': content})
    print(f'  ✓ {fn}')
"
```

至于演示时新建的 Case 数据，可保留不清（演示历史 case 在「Case 看板」可直接重新打开报告页查看）。
